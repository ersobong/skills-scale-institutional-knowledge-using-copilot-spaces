# OctoAcme Roles & Interactions Matrix

This matrix maps each role to key artifacts and ceremonies across the project lifecycle using a RACI-like ownership model.

**Key:** R = Responsible · A = Accountable · C = Consulted · I = Informed

---

## Artifacts

| Artifact | Project Manager | Product Manager | Developers | Scrum Master | UX/UI Designer | Data Analyst | Stakeholder Champion |
|---|:---:|:---:|:---:|:---:|:---:|:---:|:---:|
| Project Charter / One-pager | A | R | C | C | C | C | C |
| Roadmap & Release Plan | C | A/R | C | C | C | C | I |
| Sprint/Iteration Backlog | C | A/R | C | R | C | I | I |
| Acceptance Criteria | I | A | R | C | R | C | C |
| Design Specs & Prototypes | I | C | C | I | A/R | I | I |
| Risk Register | A/R | C | C | C | I | C | C |
| Metrics Dashboard | I | A | I | I | C | R | I |
| Release Notes | A | C | R | I | C | C | C |
| Retrospective Actions | A | C | R | R | C | C | C |
| Stakeholder Update/Briefing | C | C | I | I | I | I | A/R |

## Notes

- "A/R" means the same person is both accountable and primarily responsible for that item.
- Roles marked "I" are notified of outcomes but are not expected to attend every meeting.
- This matrix is a guide; actual ownership should be confirmed at project kickoff and recorded in the Project Charter.
- See [`octoacme-roles-and-personas.md`](octoacme-roles-and-personas.md) for full role descriptions and collaboration details.
