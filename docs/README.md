# OctoAcme Project Management Docs

A navigational guide and executive summary for OctoAcme's project management process.

## Overview

OctoAcme uses a lightweight, repeatable lifecycle to deliver projects predictably and transparently:

- **Lifecycle:** Initiation → Planning → Execution → Release → Retrospective/Continuous Improvement
- **Key workflows:** project one-pager/charter to kick off; prioritized backlog with acceptance criteria and a Definition of Done; project board states (Backlog → Ready → In Progress → In Review → QA → Done); small PRs linked to issues with CI tests/lint and at least one approval before merge.
- **Roles:** Project Manager (delivery coordination, schedule, risks, comms), Product Manager/Lead (outcomes, prioritization, success metrics), Developers (build/test/docs, estimates, risk mitigation), QA/Testing (validate acceptance criteria), Stakeholders/Sponsors (inputs and approvals).
- **Communication:** daily standups, weekly delivery syncs, sprint/milestone demos, weekly status updates with a standard template, a single source of truth for project status, and a structured escalation path (Team → PM → Product Lead → Sponsor).
- **Quality assurance:** unit and integration tests, end-to-end smoke tests pre-release, security scanning in CI, manual QA when needed; releases require a checklist covering release notes, rollback plan, and post-deploy verification.
